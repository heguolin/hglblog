# 🌸 二次元毛玻璃博客 — 全栈开发指南

> **技术栈**：Vue 3 + Vite + Tailwind CSS v4 + GSAP ｜ NestJS + Prisma + PostgreSQL
> **原型参考**：[xinghuisama.top](https://www.xinghuisama.top/)
> **部署方案**：全部署在一台 Linux 云服务器（Nginx + Docker + SSL）
> **域名**：主站 `hgl123.icu` ｜ 图床 `img.hgl123.icu`（已部署）

---

## 一、项目全景

### 1.1 我们要做什么

一个二次元风格的个人博客，核心特色：

- **极致毛玻璃**（Glassmorphism）— 所有卡片/面板都是半透明模糊 + 光泽边框
- **Bento Grid 布局** — 首页用大小不一的卡片像便当盒一样拼接
- **丰富的视觉特效** — 弹幕背景、樱花飘落、鼠标光晕、渐变光球、打字机动画
- **完整的内容管理** — 博主通过 /admin 管理后台在浏览器里写文章、发杂谈、管理照片

### 1.2 域名与服务架构

```
同一台 Linux 云服务器
│
├── hgl123.icu (博客主站)
│   │
│   │  Nginx (:443 SSL)
│   ├── /              → 前端静态文件 (Vue 3 build 产物)
│   ├── /admin/*       → 同上（SPA 路由，管理后台页面）
│   └── /api/*         → proxy_pass → NestJS 容器 (:4000)
│                           ├── PostgreSQL 容器 (Docker 内网)
│                           └── Redis 容器 (Docker 内网)
│
└── img.hgl123.icu (图床，已部署运行)
    └── 博客所有图片存储在这里，数据库只存 URL
```

### 1.3 权限模型

| 角色 | 能做什么 | 不能做什么 |
|------|---------|-----------|
| **游客** | 浏览所有页面、看文章/杂谈/照片/友链、申请友链 | 任何写操作、访问管理后台 |
| **博主 (ADMIN)** | 上面所有 + 登录 /admin 后台、发布/编辑/删除文章、发杂谈、管理相册照片、审核友链、修改站点配置 | — |

技术实现：后端所有 GET 接口公开，所有 POST/PUT/DELETE 接口加 JWT AuthGuard + RolesGuard(ADMIN)。

### 1.4 图片管理策略

所有图片统一通过你已有的图床 `img.hgl123.icu` 管理：

```
博主在后台上传图片
    → 前端调用图床 API 上传到 img.hgl123.icu
    → 图床返回图片 URL (如 http://img.hgl123.icu/2026/06/abc.png)
    → URL 存入博客数据库
    → 前端用 <img :src="url"> 展示
```

博客服务器本身不存储任何图片文件，只存 URL。

### 4.3 API 设计

**公开接口（游客可访问）：**
```
GET  /api/health
GET  /api/posts(?page,limit,category,tag)
GET  /api/posts/recent(?limit)
GET  /api/posts/:slug                        # viewCount++
GET  /api/categories
GET  /api/tags
GET  /api/chatters(?tag)
GET  /api/chatters/:id
GET  /api/albums
GET  /api/albums/:id                         # 含 photos 列表
GET  /api/friends                            # 只返回 approved
GET  /api/site/stats
GET  /api/site/config/:key
POST /api/friends/apply                      # 任何人可申请

**需要登录 (JWT + ADMIN) 的接口：**
```
POST   /api/auth/login                       # → { token, user }
POST   /api/posts                            # 创建文章
PUT    /api/posts/:id                        # 更新文章
DELETE /api/posts/:id                        # 删除文章
POST   /api/chatters                         # 发杂谈
DELETE /api/chatters/:id
POST   /api/albums                           # 创建相册
PUT    /api/albums/:id
DELETE /api/albums/:id
POST   /api/photos                           # 添加照片(存URL)
DELETE /api/photos/:id
PUT    /api/friends/:id                      # 审核友链
DELETE /api/friends/:id
PUT    /api/site/config/:key                 # 修改配置
```

### 阶段一：项目初始化

**目标**：前后端都能跑起来，能互相通信。

**前端：**
1. `npm create vite@latest frontend -- --template vue-ts`
2. 安装依赖：vue-router, pinia, axios, tailwindcss, @tailwindcss/vite, gsap, dayjs, @vueuse/core, markdown-it, highlight.js
3. 配置 vite.config.ts：`@` 路径别名、Tailwind 插件、开发代理 `/api → localhost:4000`
4. 配置 style.css：`@import "tailwindcss"` + `@theme {}` 自定义色系 + `.glass` / `.glass-strong` 全局毛玻璃类
5. 注册所有路由（含 /admin/* 路由组），组件用空占位
6. 创建 Pinia store：app store（全局状态）+ auth store（token / user / login / logout）
7. 封装 Axios：baseURL、请求拦截器自动带 JWT、响应 401 自动跳登录页

**后端：**
1. `npx @nestjs/cli new backend`
2. 安装依赖：prisma, @prisma/client, @nestjs/config, @nestjs/jwt, @nestjs/passport, passport-jwt, bcryptjs, class-validator, class-transformer
3. `npx prisma init`，编写完整 schema.prisma
4. 创建 docker-compose.yml（PostgreSQL 16 + Redis 7）
5. `docker compose up -d postgres` → `npx prisma migrate dev`
6. 创建全局 PrismaModule / PrismaService
7. main.ts 配置 CORS、ValidationPipe、端口 4000
8. 写 GET /api/health 测试接口

**✅ 验收**：前端 :3000 显示页面，请求 /api/health 返回数据。

### 阶段四：后端 API + 认证

**目标**：全部接口写完，读写权限分离。

**先做认证模块：**
1. auth 模块：POST /api/auth/login → 验证用户名密码 → 返回 JWT token + user 信息
2. JWT 策略：从 Authorization 头解析 token
3. AuthGuard：验证 token 有效性
4. RolesGuard + @Roles('ADMIN')：校验用户角色
5. seed.ts：创建默认管理员（role=ADMIN，密码 bcrypt 哈希）

**再做业务模块（每个含 Module + Controller + Service + DTO）：**
1. posts：GET 公开 / POST+PUT+DELETE 加 @UseGuards(AuthGuard, RolesGuard)
2. chatters：同上
3. albums + photos：相册 CRUD + 照片 URL 管理
4. friends：GET 公开 / POST apply 公开 / PUT 审核加守卫
5. site：统计公开 / 配置修改加守卫
6. categories + tags：GET 公开

**每个模块的开发顺序**：schema 确认 → Module 注册 → DTO 校验 → Service 逻辑 → Controller 路由 → Postman 测试

**✅ 验收**：GET 接口无 token 可访问；POST/PUT/DELETE 不带 token 返回 401；带 ADMIN token 成功。种子数据可查。

### 阶段九：管理后台

**目标**：博主能在浏览器里管理所有内容。游客无法访问。

#### 认证流程

```
游客访问 /admin/*  →  路由守卫检测无 token  →  重定向 /admin/login
博主登录          →  POST /api/auth/login   →  JWT 存入 Pinia + localStorage
后续请求          →  Axios 拦截器自动带 token →  后端验证 → 允许
token 过期        →  401 → 自动清 token      →  跳回登录页
```

#### 要做的页面

## 六、注意事项

### 6.1 开发顺序原则

- **先整体后局部**：先搭骨架（路由+布局），再填内容
- **先 mock 后对接**：每个页面先用写死的数据做 UI，确认效果后再接 API
- **先桌面后移动**：先调好桌面端，再做响应式

### 6.2 毛玻璃的坑

- `backdrop-filter` 叠加层数不超过 3 层，否则严重卡顿
- 父元素有 `overflow:hidden` / `transform` / `filter` 时会导致 `backdrop-filter` 失效
- 加 `will-change: backdrop-filter` 提示 GPU 加速

### 6.3 安全

- **数据库密码 / JWT 密钥**：用 `openssl rand -hex 32` 生成随机强密码
- **防火墙**：只开 22/80/443，PostgreSQL 5432 绝不暴露外网
- **SSH**：建议禁用密码登录，只用密钥
- **.env 文件**：加入 `.gitignore`，不提交 Git
- **图床备份**：img.hgl123.icu 的数据也要定期备份，图床挂了博客图片会全裂
- **数据库备份**：写 cron 每天自动 pg_dump 到本地或对象存储

### 6.4 Git 分支

└── dev    ← 日常开发

    ├── feature/post-system
    ├── feature/admin-panel

```
